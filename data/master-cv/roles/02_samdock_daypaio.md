## Interview Preparation Notes

### Flagship Story: Event-Sourced CRM

1. **Why event sourcing for CRM?**
   - Audit trail is critical for CRM (who changed what when)
   - Temporal queries (state at any point in time)
   - Event replay for debugging and analytics
   - CQRS enables optimized read models

2. **Event storming process**
   - Orange stickies for domain events
   - Blue stickies for commands
   - Yellow stickies for aggregates
   - How stakeholders participated

3. **Technical trade-offs**
   - Eventual consistency handling
   - Projection rebuild strategy
   - Event schema evolution
   - Multi-tenant event store design
