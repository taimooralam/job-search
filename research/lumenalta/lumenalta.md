## GAP ANALYSIS: REACT vs ANGULAR

### The Gap

|                      | React (They Want)           | Angular (You Have)        |
| -------------------- | --------------------------- | ------------------------- |
| **Component Model**  | Functional + Hooks          | Class-based + Decorators  |
| **State Management** | useState, useContext, Redux | Services, RxJS, NgRx      |
| **Templating**       | JSX (JS + HTML mixed)       | Templates (HTML separate) |
| **Data Flow**        | One-way (props down)        | Two-way binding           |
| **Ecosystem**        | Pick your own libs          | Batteries included        |

### Why This Gap Is Smaller Than It Looks

1. **Same mental models**: Component-based UI, state management, lifecycle, routing — identical concepts
2. **TypeScript is shared**: Both use TypeScript; your TS skills transfer 100%
3. **Node.js backend is identical**: React/Angular only affects frontend; backend is unchanged
4. **You learn fast**: With 11 years experience, picking up React syntax takes 2-4 weeks, not months

### How to Address in Interview

**If they ask "Do you have React experience?"**

> "My frontend experience is primarily with Angular, which shares the same component-based architecture and TypeScript foundation. The conceptual overlap is significant — state management, routing, lifecycle hooks, testing patterns. I've picked up new frontend frameworks before, and with my TypeScript depth, I'd be productive in React within a few weeks. My strength is in architecture and Node.js, which is where the tech lead role has the biggest impact."

**Key framing:**

- Don't apologize — **reframe as transferable skills**
- Emphasize **backend strength** (Node.js) — that's 50% of fullstack
- Emphasize **leadership** — tech leads spend more time on architecture than writing React components
- Show **learning agility** — you've picked up technologies before

### React Crash Course (If Asked Technical Questions)

| React Concept  | Angular Equivalent        | Quick Translation             |
| -------------- | ------------------------- | ----------------------------- |
| `useState`     | `this.state` / Services   | Local component state         |
| `useEffect`    | `ngOnInit`, `ngOnDestroy` | Lifecycle side effects        |
| `useContext`   | Dependency Injection      | Share state across components |
| `props`        | `@Input()`                | Pass data to children         |
| `JSX`          | Templates                 | Markup syntax                 |
| `Redux`        | NgRx                      | Global state management       |
| `React Router` | Angular Router            | Navigation                    |
